# Pinned runtime dependencies for the ghcr.io/subnet112/minotaur-validator
# image (api + validator + miner processes).
#
# All versions are == (not >=) so rebuilds are reproducible and a silent
# transitive upgrade can't break consensus compatibility between leader
# and peers (signature canonicalization, EIP-712 encoding, etc).
#
# Bumping any pin requires a rebuild + `make test-unit` + a deployment
# soak on staging before promoting to :stable.
#
# Cryptographic hash-locked install via requirements.lock — generated
# (and refreshed when a pin in this file changes) with:
#     pip-compile --generate-hashes --allow-unsafe \
#         --output-file minotaur_subnet/requirements.lock \
#         minotaur_subnet/requirements.txt
# The Dockerfile prefers requirements.lock when present and falls back to
# this file with --no-deps otherwise. The hash-locked variant closes the
# residual supply-chain window where a compromised PyPI mirror could swap
# a wheel. --allow-unsafe is required so setuptools/wheel are pinned too.
